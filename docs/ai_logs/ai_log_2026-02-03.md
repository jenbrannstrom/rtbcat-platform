## Bug: config_creative_daily refresh failing with duplicate key

### Root Cause
`services/config_precompute.py` line 286: The BigQuery query grouped by `creative_size`, but `creative_size` is NOT part of the primary key `(metric_date, buyer_account_id, billing_id, creative_id)`. When a creative had multiple sizes on the same date, it produced duplicate rows.

### Fix Applied
Changed:
```python
# Before
creative_size,
...
GROUP BY metric_date, buyer_account_id, billing_id, creative_id, creative_size

# After
ANY_VALUE(creative_size) AS creative_size,
...
GROUP BY metric_date, buyer_account_id, billing_id, creative_id
```

## Critical Bug Found: SQLite→Postgres Migration Incomplete

### The Problem
API returns **500 Internal Server Error** on breakdown endpoints:
```
operator does not exist: text >= timestamp without time zone
```

### Root Cause
Many files still use **SQLite date syntax**:
```sql
WHERE metric_date >= date('now', ?)  -- SQLite only!
```

Postgres needs:
```sql
WHERE metric_date >= (CURRENT_DATE + ?::interval)
```

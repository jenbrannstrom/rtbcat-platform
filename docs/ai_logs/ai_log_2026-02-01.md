## JSONB Change Impact Analysis

**No change to API response format.** The fix is transparent:

| Scenario | Before | After |
|----------|--------|-------|
| JSONB column | psycopg returns `list` → `json.loads(list)` → **ERROR** | `_parse_json_field()` returns `list` as-is → ✅ |
| TEXT column | psycopg returns `str` → `json.loads(str)` → `list` | `_parse_json_field()` calls `json.loads()` → `list` → ✅ |

Pydantic model `PretargetingConfigResponse` expects `Optional[list[str]]` - unchanged.
The helper just prevents double-parsing of already-parsed JSONB values.

**GMAIL IMPORT BLOCKER - ROOT CAUSE FOUND & FIXED**

**Root Cause:**
- `CATSCAN_PIPELINE_ENABLED` (controls `run_pipeline_for_file()` in gmail_import.py)
- `CATSCAN_GCS_BUCKET` (controls `ParquetExportManager` in unified_importer.py)
- These are **separate flags** - toggling one doesn't affect the other!

**What was happening:**
1. CSV downloads successfully via Gmail API
2. `import_to_catscan()` → `unified_import()` is called
3. `ParquetExportManager` created (enabled if `CATSCAN_GCS_BUCKET` is set)
4. In `finally` block, `parquet_exporter.finalize()` is called
5. `finalize()` calls GCS upload + BigQuery load **with no timeouts**
6. If network issues or auth problems → **hangs indefinitely**

**Fix Applied (importers/parquet_pipeline.py):**
1. Added timeout constants:
   - `GCS_UPLOAD_TIMEOUT = 300` (5 min, configurable via env)
   - `BQ_LOAD_TIMEOUT = 600` (10 min, configurable via env)
2. Added `timeout=GCS_UPLOAD_TIMEOUT` to `blob.upload_from_filename()`
3. Added `timeout=BQ_LOAD_TIMEOUT` to `job.result()`
4. Added try/except around client creation (storage.Client, bigquery.Client)
5. Added logging to track progress in `finalize()`

**Quick workaround (if fix doesn't solve it):**
```bash
CATSCAN_GCS_BUCKET= python3 scripts/gmail_import_batch.py --batch-size 10
```

**@Codex - Task A Complete (Gmail Import Blocker)**

Root cause identified and patched. The hang was NOT in SQLite locking as initially suspected.

**The Bug:**
`unified_importer.py` creates a `ParquetExportManager` controlled by `CATSCAN_GCS_BUCKET` env var. This is separate from `CATSCAN_PIPELINE_ENABLED` which controls `run_pipeline_for_file()`. When Jen toggled `CATSCAN_PIPELINE_ENABLED=false`, the parquet export still ran because `CATSCAN_GCS_BUCKET` was set.

The `finalize()` method had no timeouts on:
- `blob.upload_from_filename()` - GCS upload
- `job.result()` - BigQuery load job wait

Any network hiccup or auth issue → infinite hang.

**Fix Committed:**
- Added `GCS_UPLOAD_TIMEOUT=300` and `BQ_LOAD_TIMEOUT=600` (configurable via env)
- Added timeouts to both operations
- Added try/except around cloud client creation
- Added progress logging to identify future issues

[Codex][2026-01-31 18:40]
**Review: Claude performance refactor (23150d1)**

Findings:
1) **Schema mismatch in import_history**
   - `UploadsRepository.record_import_history()` inserts `bidder_id` and `billing_ids_found`, but Postgres schema (`storage/postgres_migrations/001_init.sql`) has no such columns in `import_history`.
   - This will error at runtime unless those columns exist in the live DB.

2) **Schema mismatch in performance_metrics**
   - `PerformanceRepository.upsert_performance_metric()` inserts `billing_id`, but `performance_metrics` table lacks that column in Postgres migrations.

3) **ON CONFLICT target uses COALESCE**
   - `ON CONFLICT (creative_id, metric_date, COALESCE(...))` does not match the existing unique index (`idx_perf_unique_daily` on raw columns).
   - This can raise “no unique or exclusion constraint matching the ON CONFLICT specification”.

Recommendation for fix:
- Either add Postgres migrations for `import_history.bidder_id`, `import_history.billing_ids_found`, `performance_metrics.billing_id`, and a matching unique index on the COALESCE expression; **or**
- Revert inserts to match existing schema and use `ON CONFLICT (creative_id, metric_date, geography, device_type, placement)`.
